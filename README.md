# Учет НЗП (UchetNZP)

Веб‑система для учета НЗП, маршрутов, перемещений, складских операций и учета металла (поступление, остатки, планы/отчеты раскроя).

## Состав модулей (проекты решения)

Решение `UchetNZP.sln` состоит из 6 модулей:

1. **UchetNZP.Web** — ASP.NET Core MVC UI, контроллеры, страницы, экспорт (Excel/PDF), фоновые сервисы.
2. **UchetNZP.Application** — прикладная логика и сервисы (WIP, маршруты, импорт, отчеты, трансферы, раскрой).
3. **UchetNZP.Domain** — доменные сущности и enum’ы.
4. **UchetNZP.Infrastructure** — `DbContext`, миграции EF Core, инфраструктурный доступ к БД.
5. **UchetNZP.Shared** — общие утилиты и вспомогательные классы.
6. **UchetNZP.Application.Tests** — unit/integration тесты для сервисов и web-слоя.

---

## Быстрый старт

### 1) Требования

- .NET SDK 8.0+
- PostgreSQL 14+

### 2) Настройка подключения к БД

В `UchetNZP.Web/appsettings.json` (или `appsettings.Development.json`) заполните строку:

```json
"ConnectionStrings": {
  "Default": "Host=...;Port=5432;Database=...;Username=...;Password=..."
}
```

### 3) Запуск приложения

```bash
dotnet restore
dotnet build UchetNZP.sln
dotnet run --project UchetNZP.Web/UchetNZP.Web.csproj
```

При старте приложение:
- применяет миграции EF Core;
- проверяет/добавляет служебные колонки в БД (в т.ч. по аудиту и раскрою);
- выполняет базовое заполнение справочников (материалы/нормы при необходимости).

### 4) Запуск тестов

```bash
dotnet test UchetNZP.sln
```

## Инструкции по всем модулям

## 1. Модуль `UchetNZP.Web`

**Назначение:** UI и HTTP-эндпоинты.

### Что внутри
- `Controllers/` — контроллеры по разделам (склад, НЗП, маршруты, отчеты, металл, раскрой).
- `Views/` — Razor-представления.
- `Models/` — ViewModel’и.
- `Services/` — сервисы экспорта и web-зависимые службы.
- `Program.cs` — регистрация DI, middleware, миграции/инициализация.

### Как расширять
- Новый экран: `Controller + View + ViewModel`.
- Новый экспорт: добавить сервис в `Services`, зарегистрировать в DI в `Program.cs`.
- Новый endpoint API: добавить action в соответствующий контроллер.

### Важное
- Логика доступа к данным выполняется через `AppDbContext` и сервисы Application-слоя.
- Не дублировать бизнес-правила во View: выносить в `UchetNZP.Application`.

---

## 2. Модуль `UchetNZP.Application`

**Назначение:** бизнес-логика.

### Что внутри
- `Abstractions/` — интерфейсы сервисов.
- `Services/` — реализации:
  - операции НЗП (приход/расход/перемещения/остатки),
  - управление маршрутами,
  - импорт,
  - отчеты,
  - расчет металла и планы раскроя.

### Как расширять
- Добавьте интерфейс в `Abstractions/`.
- Реализацию — в `Services/`.
- Зарегистрируйте сервис в DI (`UchetNZP.Web/Program.cs`).
- Покройте изменения тестами в `UchetNZP.Application.Tests`.

### Важное
- Этот модуль должен быть независимым от UI.
- Бизнес-валидации и инварианты — здесь, а не в контроллерах.

---

## 3. Модуль `UchetNZP.Domain`

**Назначение:** доменная модель.

### Что внутри
- `Entities/` — сущности предметной области:
  - НЗП, ярлыки, движения, склад,
  - металл, поступления, потребление,
  - требования/планы/отчеты раскроя и т.д.

### Как расширять
- Добавляйте/изменяйте сущности и enum’ы в соответствии с предметной областью.
- После изменений синхронизируйте `DbContext` и миграции в `Infrastructure`.

### Важное
- Не добавлять сюда web-зависимости.
- Сохранять простую и чистую доменную модель.

---

## 4. Модуль `UchetNZP.Infrastructure`

**Назначение:** работа с БД.

### Что внутри
- `Data/AppDbContext.cs` — EF Core контекст.
- `Migrations/` — миграции схемы PostgreSQL.
- `AppDbContextFactory.cs` — фабрика для design-time миграций.

### Как работать с миграциями

```bash
dotnet ef migrations add <MigrationName> --project UchetNZP.Infrastructure --startup-project UchetNZP.Web
dotnet ef database update --project UchetNZP.Infrastructure --startup-project UchetNZP.Web
```

### Важное
- Все изменения структуры БД вносятся через миграции.
- Для production-переходов держите миграции в репозитории и применяйте последовательно.

---

## 5. Модуль `UchetNZP.Shared`

**Назначение:** общие переиспользуемые компоненты.

### Что внутри
- форматтеры и utility-классы, не относящиеся строго к domain/application/web.

### Когда использовать
- когда код нужен в нескольких модулях;
- когда это чистая утилита без бизнес-состояния.

---

## 6. Модуль `UchetNZP.Application.Tests`

**Назначение:** проверка бизнес-логики и web-поведения.

### Что внутри
- тесты сервисов `Application/Services`;
- тесты контроллеров и web-утилит.

### Правила
- на каждое новое бизнес-правило — минимум один позитивный и один негативный тест;
- при исправлении багов — регрессионный тест на сценарий ошибки.

---

## Частая ошибка и как избежать

Ошибка вида:

`PostgresException: 42703: столбец ... не существует`

Обычно означает рассинхронизацию кода и БД (миграции не применены или схема «дрейфовала»).

Что делать:
1. Проверить актуальность строки подключения.
2. Выполнить миграции (`dotnet ef database update ...`).
3. Перезапустить приложение.
4. Убедиться, что стартовая инициализация в `Program.cs` отработала без исключений.
