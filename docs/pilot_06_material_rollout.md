# Пилот по подбору материалов 06.xxxx и раскрою

## 1) Пилотные позиции (реальные номенклатуры)

Для пилота берем 10 позиций (2 группы «пруток/круг» и 2 группы «лист»):

- **Штыри:** Ø8×17, Ø8×23, Ø8×26, Ø8×29.
- **Бирки:** 7×12, 15×25, 15×65, 5×36.
- **Пластины/наконечники:** 1,5×38×90 и 2×55×30.

Рекомендуемая запись в нормах (поле `SizeRaw`) для корректного парсинга и авто-подбора:

- `Ø8x17`, `Ø8x23`, `Ø8x26`, `Ø8x29`
- `7x12`, `15x25`, `15x65`, `5x36`
- `1.5x38x90`, `2x55x30`

## 2) Связка с материалами 06.xxxx (3–5 шт.)

Набор для пилота (5 материалов, чтобы покрыть и пруток, и лист):

1. `06.1006` — круг/пруток Ø8 (базовый для штырей).
2. `06.1008` — круг/пруток Ø10 (резервный для штырей при дефиците Ø8).
3. `06.2001` — лист 1.5 мм (базовый для бирок и пластин 1,5 мм).
4. `06.2003` — лист 2.0 мм (базовый для наконечников 2 мм).
5. `06.2004` — лист 2.5 мм (резервный листовой материал).

Правило соответствия для пилота:

- детали с шаблоном имени `штыр` + геометрия `rod` + размер 7–9 мм → `06.1006`;
- детали с шаблоном имени `штыр` + геометрия `rod` + размер 9–11 мм → `06.1008`;
- детали с шаблоном имени `бирк` + геометрия `sheet` + толщина 1.0–1.6 мм → `06.2001`;
- детали с шаблоном имени `пластин|наконеч` + геометрия `sheet` + толщина 1.7–2.2 мм → `06.2003`;
- fallback для `sheet` при отсутствии основного материала → `06.2004`.

## 3) Сквозная цепочка пилота

### Шаг A. Импорт

1. В **Маршруты → Импорт** загрузить Excel с листами `materials` и `norms`.
2. Сначала сделать preview (`/routes/import/metal-data/preview`), затем execute (`/routes/import/metal-data/execute`).
3. Проверить, что по пилотным позициям парсер размера отдал корректные `ShapeType/ThicknessMm/DiameterMm/LengthMm`.

### Шаг B. Подбор материала

1. Создать запуск партии по пилотной детали.
2. Нажать создание требования металла из запуска.
3. На требовании убедиться, что `SelectionSource=auto_rule` для основных сценариев.
4. Если `fallback/manual`, фиксируем причину в журнал пилота.

### Шаг C. Раскрой

1. В `MetalWarehouse/RequirementDetails` запустить генерацию карты раскроя:
   - для стержней использовать длину хлыста и резы под 17/23/26/29;
   - для листов учитывать полезный выход по картам 7×12, 15×25, 15×65, 5×36, 1.5×38×90, 2×55×30.
2. Проверить `%Utilization` и `ScrapPercent` в сгенерированном плане.

### Шаг D. Требование на склад

1. Перевести требование в печатную форму кладовщика.
2. Подобрать исходные заготовки/листы со склада (`Stock`), предпочтительно с учетом FIFO и подходящих остатков.
3. Зафиксировать выбранные партии (номер прихода, код заготовки).

### Шаг E. Факт резки

1. По каждой карте раскроя внести факт выполнения (`execution status`, фактический остаток, лом).
2. Убедиться, что исходная заготовка списана, а полезный выход/остатки/лом созданы отдельными единицами.
3. Проверить, что движение отражается в `CuttingReports`.

## 4) KPI пилота (фиксируем по итогу смены/недели)

### KPI-1: Точность пересчетов

- Формула:
  - `Accuracy = 1 - |Плановый_вес - Фактический_вес| / Плановый_вес`
- Цель пилота: **не ниже 97%**.
- Отдельно считать по группам: `rod` и `sheet`.

### KPI-2: Снижение отхода

- База сравнения: средний % отхода за период до пилота (например, 4 недели).
- Формула:
  - `WasteReduction = (BaselineScrap% - PilotScrap%) / BaselineScrap%`
- Цель пилота: **снижение не менее 8–12%**.

### KPI-3: Доля повторно использованных остатков

- Формула:
  - `ReuseShare = Масса(повторно использованных остатков) / Масса(всех образованных остатков)`
- Цель пилота: **не ниже 25%** для листа и **не ниже 15%** для прутка.

## 5) Правила/коэффициенты после пилота и масштабирование

После завершения пилота утверждаем:

1. **Коэффициенты запаса по типам проката**:
   - `rod`: +1.5…2.0% к расчетной потребности,
   - `sheet`: +2.0…3.0%.
2. **Порог «делового остатка»** (минимум для повторного использования):
   - rod: от 120 мм,
   - sheet: от 40×40 мм.
3. **Приоритеты правил подбора** (`part_to_material_rule.Priority`) по фактической точности.
4. **Список обязательных кандидатов** (`CandidateMaterials`) для типовых деталей.

Масштабирование на весь справочник:

- добавляем правила пакетами по семействам деталей (штыри → бирки → пластины/наконечники → прочие листовые/прутковые);
- каждый пакет сначала идет в preview-режиме, затем в продуктив;
- критерий допуска пакета: достижение KPI пилота ±10%.

## Минимальный протокол пилота (для утверждения)

- Период пилота: даты начала/окончания.
- Количество требований в пилоте.
- KPI-1/2/3 (факт).
- Набор утвержденных коэффициентов и порогов.
- Список изменений правил (`part_to_material_rule`) перед масштабированием.
